## 1. Executive Overview

GOHSY FASHION은 유휴 GPU 인프라(dongseon-studio)를
실질적 수익 엔진으로 전환하는 프로젝트다.

단순한 의류 유통이 아니다.

```
콘텐츠 생성 → AI 합성 → 영상 변환 → 다채널 유통 → 매출 데이터 → 재기획
```

이 루프가 자동으로 돌아가는 시스템을 만든다.

핵심 자산 4개:

| 자산 | 역할 |
|------|------|
| 박씨 (parksy 생태계) | 이미지/오디오/렌더 — 생산 엔진 |
| 고씨 (비주얼 물성) | 배우 기반 실물 앵커 — 브랜드 얼굴 |
| GPU (dongseon-studio) | 대량 합성 — 생산 자동화 장치 |
| 80-90 아카이브 | 일본 남성 잡지 스타일 DB — 무한 소스 |

## 2. 전략적 정의

### 패션은 리팩토링된 언어다

스타일 = 계급 언어.
스타일링 = 과거 유산의 현대적 재조합.
상품 = 콘텐츠의 파생 결과물.

GOHSY는 "남성 패션 전체"를 다루되, 무질서한 잡화몰이 아니라:

> **"남성 스타일 리팩토링 스튜디오"**

로 정의한다.

### 왜 남성인가

- 남성 패션 콘텐츠 시장은 여성 대비 공급 부족
- 남성 구매 패턴: 탐색 시간 짧고 브랜드 충성도 높음 → 한번 잡으면 반복 구매
- 80-90 아카이브가 남성 잡지 기반 → 데이터 소스와 타겟이 일치

## 3. 타겟 고객

### 3.1 1차 타겟: "옷 잘 입고 싶은데 방법을 모르는 남자"

| 항목 | 정의 |
|------|------|
| 연령 | 25-40세 |
| 소득 | 월 250-500만원 (직장인, 자영업) |
| 특성 | 패션에 관심은 있으나 무신사/29cm 탐색이 귀찮음 |
| 구매 행동 | 검색 → 영상 확인 → 즉시 구매 (의사결정 빠름) |
| 통증점 | "뭘 사야 할지 모르겠다", "코디를 못하겠다" |

### 3.2 왜 GOHSY에서 사는가

쿠팡에 같은 옷이 수천 개 있다. 차이점:

| 일반 셀러 | GOHSY |
|-----------|-------|
| 상품 사진 1장 | 룩북 + 웹툰 + 코디 영상 |
| "이 옷 어때요" | "이 옷을 이렇게 입어라" |
| 상품 나열 | 스타일 제안 |

**판매하는 것은 옷이 아니라 "코디 솔루션"이다.**

### 3.3 2차 타겟: "복고 감성에 반응하는 남자"

- 80-90년대 일본 시티팝/아메카지 감성
- 유튜브에서 "빈티지 패션", "아메카지" 검색하는 층
- 이 층은 콘텐츠 충성도가 높고, 팬덤화 가능

## 4. 시장 분석

### 4.1 범위: 남성 패션 전 카테고리

1. 스트릿
2. 클래식/수트
3. 캐주얼
4. 워크웨어
5. 아카이브 복고

초기 집중: **캐주얼 + 아카이브 복고** (아카이브 DB와 직결되는 카테고리)

### 4.2 경쟁자 분석

| 경쟁자 | 강점 | 약점 | GOHSY 대비 |
|--------|------|------|------------|
| 무신사 | 브랜드 집합, 트래픽 | 자체 콘텐츠 약함, 남성 비중 감소 | 무신사에 없는 코디 콘텐츠 |
| 29cm | 큐레이션, 감성 | 여성 중심, 객단가 높음 | 남성 특화 포지션 |
| 쿠팡 패션 셀러 | 가격 경쟁력, 트래픽 | 콘텐츠 없음, 상품 사진만 | 콘텐츠 양 10배 차이 |
| 에이블리/지그재그 | MZ 여성 특화 | 남성 없음 | 시장 겹침 없음 |
| YouTube 패션 채널 | 구독자, 영향력 | 직접 판매 안 함 | 콘텐츠+판매 통합 |

**포지션**: 쿠팡의 가격 + 무신사의 큐레이션 + YouTube의 콘텐츠를 **1인 AI 시스템**으로 구현.

### 4.3 차별성 매트릭스

| 항목 | 일반 셀러 | GOHSY |
|------|-----------|-------|
| 상품 촬영 | 스튜디오 촬영 후 업로드 | AI 합성 대량 생성 |
| 콘텐츠 비율 | 1상품 1이미지 | 1상품 10콘텐츠 (룩북+웹툰+영상+도면) |
| 운영 방식 | 재고 중심 | 콘텐츠 중심 |
| 의사결정 | 감각 의존 | 데이터 축적 기반 |
| 인력 | 3-5인 | 1인 + AI |

## 5. 핵심 자산 통합

### 5.1 80-90 아카이브 → 스타일 구조 DB

80-90년대 일본 남성 잡지 스타일링을 AI로 분석한다.
단순 참고가 아니다. 구조를 추출한다:

| 추출 대상 | 형식 | 용도 |
|-----------|------|------|
| 레이어링 공식 | JSON 룰셋 | "이너 + 아우터 + 액세서리" 조합 규칙 |
| 컬러 매칭 패턴 | 색상 팔레트 DB | 시즌별 컬러 추천 |
| 핏 비율 데이터 | 체형별 매핑 | 사이즈 추천 |
| 소품 배치 구조 | 레이아웃 규칙 | 촬영/합성 시 자동 배치 |

이 DB가 쌓이면 **"80년대 아메카지 스타일로 키 175 보통 체형 봄 코디"** 같은 쿼리에
자동으로 룩북을 생성할 수 있다.

**현재 상태: 미구축. Phase 1에서 10종 파일럿으로 시작.**

### 5.2 AI 생산 파이프라인

| 단계 | 도구 | 역할 |
|------|------|------|
| 이미지 생성 | parksy-image/scripts/render/ | 룩북, 착용샷, 상세컷 |
| 웹툰 생성 | parksy-image/scripts/story/ | 캐릭터 착용 에피소드 |
| CAD 도면 | parksy-image/scripts/cad/ | 패턴, 스펙시트 |
| 영상 변환 | gpu/ + render.html | 상품 소개 영상 |
| BGM | parksy-audio/lyria3/ | 브랜드 사운드 |
| 모션 클립 | 외부 AI (추후 선정) | 3초 움직임 |

**생산 비용 구조:**

| 항목 | 기존 촬영 | GOHSY AI |
|------|-----------|----------|
| 모델 섭외 | 50-200만/일 | 0원 (AI 합성 + 고씨 앵커) |
| 스튜디오 대여 | 30-50만/일 | 0원 (배경 합성) |
| 사진작가 | 30-50만/일 | 0원 |
| 편집/보정 | 10-30만/건 | API 비용 (건당 수백원) |
| **SKU당 비용** | **30-50만원** | **1,000-5,000원** |

기존 대비 **99% 비용 절감**. "0원"이 아니라 "99% 절감"이다.
API 호출, 큐레이션 시간, 품질 검수 시간은 0이 아니다.

### 5.3 고씨 비주얼 자산

AI 합성 모델은 누구나 만든다.
하지만 **배우 기반 실물 앵커**는 다르다.

- AI 이미지만으로는 신뢰 부족 → 실물 사진이 섞여야 전환율 올라감
- 고씨 = 브랜드 얼굴. 일관된 비주얼 아이덴티티
- AI 모델 + 실물 앵커 혼합 = "이 브랜드는 실체가 있다"

## 6. 채널 구조: 4채널 루프

```
PRODUCT_CARD.json (유일한 수동 입력)
       │
       ▼
   AI 에셋 생성 (parksy 생태계)
       │
       ▼
   GPU 렌더링 (상세페이지 + 영상)
       │
       ├──→ ① 쿠팡 (Revenue)
       ├──→ ② 네이버 스마트스토어 (Revenue)
       ├──→ ③ YouTube (Traffic)
       └──→ ④ gohsy.com (Brand)
                │
                ▼
           매출 데이터 수집
                │
                ▼
           다음 PRODUCT_CARD (루프백)
```

### 6.1 쿠팡 — 매출 엔진

- 한국 이커머스 최대 트래픽
- 로켓배송 신뢰 → 전환율 높음
- 위탁/마켓플레이스로 재고 부담 최소화
- **역할: 돈을 버는 곳**

### 6.2 네이버 스마트스토어 — 검색 유입

- 한국 쇼핑 검색의 50%+ = 네이버
- 네이버 쇼핑 상위 노출 → 자연 유입
- 공식 API 제공 → GUI 자동화보다 안정적
- **역할: 검색으로 고객을 잡는 곳**

### 6.3 YouTube — 트래픽 엔진

4개 시리즈:

| 시리즈 | 포맷 | 빈도 |
|--------|------|------|
| A: "옷 입는 웹툰" | 웹툰 에피소드 3-5분 | 주 2회 |
| B: "도면에서 옷장까지" | CAD 타임랩스 쇼츠 1-3분 | 주 1회 |
| C: "판타지 룩북" | 세계관 시네마틱 2-5분 | 시즌당 1회 |
| D: "실착 리뷰" | 브이로그/쇼츠 1-10분 | 상품 출시마다 |

영상 설명에 쿠팡/네이버/gohsy.com 링크 삽입.
24시간 루핑 채널 운영으로 노출 극대화.

**역할: 사람을 데려오는 곳**

### 6.4 gohsy.com — 브랜드 쇼룸

- YouTube 트래픽 착지 페이지
- 룩북 갤러리, 웹툰 아카이브, 판타지 월드
- 쿠팡/네이버 구매 리다이렉트 허브
- 매출 추적 대시보드 (비공개)

**역할: 세계관을 보여주는 곳**

## 7. 수익 모델

### 7.1 초기: 위탁/POD (재고 0)

| 항목 | 구조 |
|------|------|
| 소싱 | 위탁 판매 (동대문/도매) 또는 POD |
| 재고 | 0 — 주문 후 발주 |
| 마진 | 위탁 15-25%, POD 30-40% |
| 리스크 | 최소 (재고 부담 없음) |

위탁 판매가 "장사"로 보일 수 있다.
하지만 **검증 단계에서 재고를 쌓는 건 자살 행위**다.
데이터가 쌓인 후에 자체 기획으로 전환한다.

### 7.2 확장: 자체 셀렉션

| 조건 | 전환 기준 |
|------|-----------|
| 전환 시점 | 월 100건 이상 판매 SKU 확인 후 |
| 자체 기획 | 판매 데이터 기반 + 아카이브 DB 조합 |
| 마진 구조 | 원가 20-30% → 소비자가 대비 마진 50-60% |
| 유통 | 고씨 거점 활용 직접 유통 가능 |

### 7.3 수치 시뮬레이션

**전제 조건 (보수적):**

| 항목 | 값 | 근거 |
|------|-----|------|
| SKU | 30개 (초기) | 위탁 기준 관리 가능 수량 |
| 객단가 | 39,000원 | 캐주얼 남성복 평균 |
| 일 트래픽 | 500 (쿠팡+네이버 합산) | 신규 셀러 초기 3개월 평균 |
| 전환율 | 0.8% | 신규 셀러 초기 (업계 평균 2% 대비 보수적) |
| 마진율 | 20% | 위탁 기준 |

```
일 주문: 500 × 0.008 = 4건
일 매출: 4 × 39,000 = 156,000원
월 매출: 156,000 × 30 = 4,680,000원
월 순이익: 4,680,000 × 0.20 = 936,000원
```

**6개월 후 (트래픽 + SKU 확장):**

| 항목 | 값 |
|------|-----|
| SKU | 100개 |
| 일 트래픽 | 2,000 (YouTube 유입 포함) |
| 전환율 | 1.2% (리뷰 축적 효과) |
| 객단가 | 45,000원 (자체 기획 혼합) |
| 마진율 | 30% (자체 기획 비중 증가) |

```
일 주문: 2,000 × 0.012 = 24건
월 매출: 24 × 45,000 × 30 = 32,400,000원
월 순이익: 32,400,000 × 0.30 = 9,720,000원
```

**이 수치는 매월 실제 데이터로 교체한다. 가정은 가정일 뿐이다.**

### 7.4 손익분기점

| 항목 | 값 |
|------|-----|
| 월 고정비용 | 약 50만원 (도메인, API 호출, 기타) |
| 손익분기 매출 | 약 250만원/월 (마진 20% 기준) |
| 도달 예상 시점 | 운영 2-3개월차 |

## 8. 방어벽 (Defensibility)

AI 자체는 방어벽이 아니다. 누구나 쓸 수 있다.

### 8.1 진짜 방어벽 3개

**① 스타일 구조 DB (80-90 아카이브)**

쿠팡이 갖고 있는 건 "구매 데이터"다.
GOHSY가 쌓는 건 "스타일 구조 데이터"다.

```
레이어링 공식 × 컬러 매칭 × 체형별 핏 × 시즌 변수
= 자동 코디 생성 엔진
```

이 DB는 시간이 갈수록 두꺼워진다. 후발 주자가 따라오려면 같은 시간을 써야 한다.

**② 생산 속도 (AI 파이프라인)**

같은 시간에 콘텐츠 10배 생산.
경쟁자가 상품 사진 1장 올릴 때 GOHSY는 룩북 + 웹툰 + 영상 + 코디 가이드를 올린다.
양의 차이가 누적되면 검색 노출, 체류 시간, 전환율 전부 차이 난다.

**③ 고씨 비주얼 앵커**

AI 합성만으로는 "가짜 느낌"을 지울 수 없다.
실물 앵커(고씨)가 섞이면 **브랜드 신뢰**가 생긴다.
이건 복제 불가. 사람은 복사가 안 된다.

### 8.2 방어벽이 아닌 것

| 착각하기 쉬운 것 | 왜 방어벽이 아닌가 |
|-------------------|---------------------|
| "AI를 쓴다" | 경쟁자도 다 쓴다 |
| "자동화했다" | 도구는 누구나 산다 |
| "GPU가 있다" | 클라우드로 빌리면 된다 |
| "쿠팡/네이버 매출 데이터" | 그들이 수억 건 갖고 있다 |

## 9. 리스크 분석

### 9.1 운영 리스크

| 리스크 | 확률 | 심각도 | 대응 |
|--------|------|--------|------|
| 상품 소싱 경험 부족 | 높음 | 중 | 위탁/POD로 시작. 직접 소싱은 데이터 확보 후 |

## 10. 병목 진단

**AI는 병목이 아니다.**

AI 파이프라인은 이미 설계됐고, 비용은 거의 없다.

진짜 병목:

```
① 물류    — 위탁이면 위탁처 품질, 자체면 3PL 계약
② 반품    — 온라인 의류 반품률 20-30%. 이걸 견딜 CS 체력
③ CS      — 1인 운영에서 CS 부하 = 시스템 멈춤
④ 정책    — 쿠팡/네이버 정책 변경에 즉각 대응하는 민첩성
⑤ 소싱    — 좋은 위탁처 확보 = 상품 품질 = 반품률 = 리뷰 = 매출
```

이걸 해결하지 못하면 GPU는 장난감이다.

**대응 원칙:**

| 병목 | 원칙 |
|------|------|
| 물류 | 위탁으로 시작 → 데이터 쌓이면 3PL |
| 반품 | SKU 30개 제한 → CS 부하 관리 가능 범위 유지 |
| CS | 자동 응답 템플릿 + FAQ 페이지 → 1인 처리 가능 구조 |
| 정책 | adapter 패턴 → 플랫폼 변경 시 adapter만 수정 |
| 소싱 | 3곳 이상 위탁처 확보 → 단일 의존 방지 |
